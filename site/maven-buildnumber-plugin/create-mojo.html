buildnumber:create

Full name:

com.adaptiweb.mojo:maven-buildnumber-plugin:0.1-SNAPSHOT:create

Description:

Goal which provide some dynamic information in build time thru project properties.
Right now are supported these four properties:
  1. buildNumberProperty
  2. timestampProperty
  3. revisionNumberProperty
  4. lastCommittedRevisionProperty

Attributes:

  • Requires a Maven 2.0 project to be executed.
  • Binds by default to the lifecycle phase: validate.

Optional Parameters

Name Type Since Description
basedir File - Working copy of project resources managed by Subverion. Default ${basedir}.
buildNumberProperty String - Name of property for build number. Default 'buildNumber'.
doCheck boolean - Check if files in working copy are up to date and commited.
Default value is: true.
excludes String[] - List of patterns to match resources excludes from SVN checking.
ignoreUnversioned boolean - If true (default) unversioned resources are ignored while checking SVN status.
Default value is: true.
includes String[] - List of patterns to match resources includes for SVN checking.
lastCommittedRevisionProperty String - Name of property for last commited revision number. No property name is set by default. This parameter is ignored if project isn't under subversion control.
neverFail boolean - Project build never fails, even if SVN check find some problems.
Default value is: true.
overwriteProperties boolean - Properties are sets even they already exists. Default false.
Default value is: false.
revisionNumberProperty String - Name of property for revision number. No property name is set by default. This parameter is ignored if project isn't under subversion control.
timestampFormat String - Format for timestamp property. Default 'Y-m-d H:M:S'.
Default value is: Y-m-d H:M:S.
timestampProperty String - Name of property for timestamp. No property name is set by default. Value of property is formated by timestampFormat parameter.

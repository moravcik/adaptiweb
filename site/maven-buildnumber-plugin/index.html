Introduction

This Maven plugin (mojo) provided some dynamic information about project at build time. These information can be used to create a unique build number. Informations are passed to project properties. Right now are supported these four informations:

Iformation Default Property Name
Configuration Tag		 Note
Build Number
(derived from project version)		 buildNumber
<buildNumberProperty>		 If project version is SNAPSHOT then SNAPSHOT is preplaced by timestamp.
Example: 1.0-20090118134045
Time-Stamp not set
<timestampProperty>		 Default timestamp format is 'Y-m-d H:M:S'
and can be changed by property <timestampFormat>.
Revision Number not set
<revisionNumberProperty>		 Only if project has Subversion as SCM.
Commited Revision Number not set
<lastCommittedRevisionProperty>		 Only if project has Subversion as SCM.

The mojo also controls status of working copy files. This can be disabled by setting property <doCheck>.

Detailed description about mojo parameter are available here.
